Project background

The problem

In today's digital landscape, a compelling online presence is crucial for frontend developers to effectively showcase their skills, projects, and professional journey. The challenge was to create a personal portfolio website that not only acted as a central repository for projects like "Omnifood," "Modern Animated Login/Signup UI," and "The Pig Game" but also demonstrated a mastery of core web technologies and design principles. The goal was to build an intuitive, visually appealing, and highly responsive platform to engage potential employers and clients.

My role

As the sole developer, I conceptualized, designed, and implemented every aspect of this portfolio website. This involved crafting the semantic HTML structure, applying extensive CSS for styling and responsiveness across various devices, and implementing interactive elements using JavaScript. My role encompassed the entire frontend development lifecycle, from initial wireframing to deployment, ensuring a cohesive and functional user experience.

Why I built this

Learning goals

This project served as a comprehensive application and reinforcement of various frontend development concepts:

  • HTML5 structure: Gaining deeper proficiency in building well-organized, semantic HTML for complex multi-page applications (e.g., index.html, about.html, projects.html, stack.html, contact.html).
  • Advanced CSS3 & responsiveness: Mastering responsive design techniques through media queries (index-mobile-queries.css, index-tablet-queries.css) to ensure optimal viewing on desktops, tablets, and mobile devices, along with advanced layout techniques like CSS Grid and Flexbox.
  • JavaScript interactivity & DOM manipulation: Implementing dynamic features such as scroll-based navigation control, project image hover effects, and interactive elements on the "Stack" page, demonstrating effective manipulation of the Document Object Model (DOM).
  • Modular web development: Practicing the organization of code into logical, reusable components (e.g., separate CSS and JS files for different sections and shared functionalities).

Personal motivation

My motivation stemmed from the desire to create a professional, polished, and personalized online portfolio that truly reflects my capabilities as a frontend developer. I aimed to build a platform that not only showcases completed projects but also demonstrates the thought process and technical skills behind their creation. This project was an opportunity to apply my knowledge in a real-world scenario, offering a tangible representation of my passion for web development.

Deep dive: How I built it

1. Architecture & tech stack

Frontend

  • HTML5 Serves as the backbone, defining the structure of the main portfolio, dedicated project pages, about me, skill stack, and contact sections. It incorporates semantic tags for clarity and SEO.
  • CSS3 EProvides extensive styling for a modern aesthetic, manages responsive layouts across various breakpoints, and implements visual effects and transitions. Separate CSS files (index.css, projects.css, stack.css, contact.css, about.css) ensure maintainability and modularity, alongside dedicated mobile and tablet query files.
  • JavaScript: Drives the interactive components, including navigation behavior, dynamic project overlays, and mobile-specific functionalities, enhancing the overall user experience.

Visual assets

  • Picture: Pictures in .webp formats have smaller sizes but the quality is not compromized.
  • Background pattern: pattern.svg for subtle visual texture.
  • Project thumbnails: my-portfolio.webp as the main thumbnail for the portfolio's own project listing.
  • Icons A comprehensive set of SVG icons (e.g., envelope-white-100.svg, arrow-right.svg, eye.svg, various social media and skill icons) for intuitive visual cues and branding.
  • Favicon: my-pic.ico for browser tab identification.

2. Core features & implementations

(A) Responsive navigation control

Challenge: Creating a navigation system that remains functional and aesthetically pleasing across all screen sizes, including dynamic hiding/showing on scroll for mobile.

Solution: Implemented a fixed navigation bar with distinct layouts for desktop, tablet, and mobile. JavaScript is used to detect scroll direction and dynamically apply CSS classes (hide-on-scroll) to the navigation bar, ensuring it hides when scrolling down and reappears when scrolling up, optimizing screen real estate on smaller devices.

(B) Dynamic project showcase

Challenge: Presenting multiple projects in an engaging and interactive grid layout on the main projects page.

Solution: Utilized CSS Grid for a flexible project container layout (all-projects-container). JavaScript listens for mouseover and mouseleave events on each project container (proj-cont). Upon hover, a view-project-overlay with an eye icon and "View project" text becomes visible, and the project image scales slightly, providing visual feedback and indicating interactivity.

(C) Reusable image gallery for project details

Challenge: Providing a consistent and interactive image gallery experience on individual project case study pages, including the "My Portfolio Website" case study itself.

Solution: A single JavaScript file (project-1.js) is reused across all project detail pages. This script handles the main project image display, updates it based on thumbnail clicks, and applies subtle hover effects to thumbnails, ensuring a uniform gallery experience without redundant code.

Challenges & solutions

Challenge: Ensuring cross-device responsiveness

Solution: Implemented a mobile-first design approach with extensive use of flexible units, media queries (index-mobile-queries.css, index-tablet-queries.css), CSS Grid, and Flexbox to create adaptive layouts that adjust seamlessly to various screen sizes.

Lesson learned: Thorough planning for responsive design from the outset saves significant refactoring time and results in a more robust and accessible website.

Challenge: Dynamic content and UI updates

Solution: Utilized JavaScript to dynamically manipulate the DOM, such as changing navigation bar visibility based on scroll, updating project image states on hover, and handling interactive elements on the "Stack" page, ensuring a smooth and engaging user experience.

Lesson learned: JavaScript-driven DOM manipulation is key for creating interactive and dynamic web applications, providing immediate visual feedback to user actions.

Challenge: Maintaining code organization & scalability

Solution: Segregated CSS into multiple files based on section (index.css, about.css projects.css, stack.css, contact.css) and responsiveness (-mobile-queries.css, -tablet-queries.css). JavaScript was also modularized where possible (e.g., project-1.js for gallery).

Lesson learned: Modular codebases are essential for larger projects, improving maintainability, readability, and scalability by breaking down complex systems into manageable components.

Accessibility (A11y)

While the primary focus was on visual design and functionality, basic accessibility considerations were integrated:

  • Semantic HTML: Proper use of HTML5 semantic elements (<nav>, <main>, <section>, <footer>, <h1> through <h6>, <button>, <a>) ensures a meaningful structure for assistive technologies.
  • Descriptive link and button text: All interactive elements (<button>, <a>) are given clear, descriptive text labels (e.g., "View all projects," "Contact me," "Send your message"), aiding users who navigate via screen readers or keyboards.
  • Alternative text for images: Important images (my-pic.webp, various icons) include alt attributes, providing textual descriptions for users who cannot see the images.
  • Keyboard navigation: Standard HTML elements naturally support keyboard navigation, allowing users to tab through interactive elements.
