Project background

The problem

As a classic dice game, "Pig Game" offers a simple yet engaging challenge for two players. The goal was to digitally recreate this game, providing an intuitive and interactive user interface that clearly displays game state, player scores, and allows for dynamic gameplay.

My role

As the sole developer, I was responsible for the entire implementation of this game. This included structuring the HTML, styling the interface with CSS, and implementing the core game logic and interactivity using JavaScript. I also leveraged a pre-designed flowchart to guide the game's functional design.

Why I built this

Learning goals

This project served as an excellent opportunity for me to solidify my understanding and practical application of:

  • Core JavaScript logic: Implementing game rules, managing state variables, and handling complex conditional flows.
  • DOM manipulation: Dynamically updating elements like scores, dice images, and player active/winner states based on game progress.
  • Event handling: Attaching event listeners to buttons (roll dice, hold, new game) to drive game actions.
  • CSS for dynamic UI: Using CSS to visually represent game states, such as active players and winners, and to hide/show the dice.

Personal motivation

I was motivated to build a complete, functional, and visually appealing interactive game from scratch. This project allowed me to apply foundational web development skills in a fun, practical context, demonstrating my ability to translate game rules into robust code.

Deep dive: How I built it

1. Architecture & tech stack

Frontend

  • HTML5 Provides the structural foundation for the game interface, including player panels, score displays, the dice image placeholder, and control buttons.
  • CSS3 Styles the game, ensuring a clean, modern look. It handles player active states, winner highlighting, and the visual appearance of the dice and buttons.
  • JavaScript: The backbone of the game, managing all game logic, player turns, score updates, dice rolls, and UI manipulation in response to user actions.

Visual assests

  • Dice images: Six PNG images (dice-1.png to dice-6.png) are dynamically loaded based on the dice roll.
  • Flowchart: A pig-game-flowchart.jpg file guided the implementation of the game logic, ensuring all rules and states were accounted for.

2. Core game logic & features

(A) Game state management

Challenge: Keeping track of global game variables like current scores, total scores, active player, and game status (playing).

Solution: I used global JavaScript variables (scores, currentScore, activePlayer, playing) that are initialized at the start of each new game via an init() function. This centralizes state management, making it easier to control game flow.

(B) Dice roll and score accumulation

Challenge: Generating random dice rolls, displaying the correct dice image, and adding the result to the current player's temporary score.

Solution: An event listener on the "Roll dice" button generates a random number (1–6). The diceEl.src is updated dynamically using template literals (dice-${dice}.png) to show the corresponding dice image. If the roll is not 1, the dice value is added to currentScore and updated on the UI.

(C) Holding score and player switching

Challenge: Allowing the active player to "hold" their current score, add it to their total, check for a win, and then switch to the next player.

Solution: The "Hold" button's event listener adds currentScore to scores[activePlayer]. A check then determines if scores[activePlayer] has reached or exceeded 100, declaring a winner by adding the player--winner class and ending the game. If no win, the switchPlayer() function is called, which resets currentScore to 0, updates the UI, and toggles the player--active class between players.

Challenges & solutions

Challenge: Managing complex game state

Solution: Centralizing game variables (scores, currentScore, activePlayer, playing) and initializing them with a dedicated init() function at game start.

Lesson learned: Clear state management is crucial for complex interactive applications.

Challenge: Dynamically updating UI based on game logic

Solution: Using textContent to update scores and classList.add/remove/toggle to change player active/winner states and dice visibility, driven by JavaScript logic.

Lesson learned: Direct DOM manipulation is effective for immediate visual feedback in simple games.

Challenge: Implementing turn-based logic (roll of 1)

Solution: Creating a switchPlayer() function that resets the current score and toggles the player--active class, ensuring smooth transitions between turns.

Lesson learned: Modular functions improve code readability and maintainability for repetitive actions.

Accessibility (A11y)

While the focus was on core game logic and UI, basic accessibility considerations include:

  • Semantic HTML: Using <h2>, <p>, and <button> elements appropriately.
  • Clear text labels: Buttons have descriptive text like "New game", "Roll dice", "Hold".
  • Alternative text for images: The dice image includes an alt attribute ("Playing dice") for screen readers.
  • Keyboard navigation: Buttons are naturally focusable and operable via keyboard.

Results & metrics

The "Pig Game" successfully delivers a fully functional and engaging two-player dice game experience. The clear visual cues for active players, current scores, and total scores make the game intuitive to play.

User feedback

Simple but addictive! The visual dice rolls are great.

I love how easy it is to start a new game and understand whose turn it is.

What I'd do differently

Reflecting on the project, I would explore the following for future enhancements:

  • Player name input: Allow users to enter their names instead of generic "Player 1" and "Player 2."
  • Winning score customization: Enable players to set a custom target score to win, rather than the fixed 100.
  • Instructions overlay: Add a simple modal or overlay to explain the game rules to new players.
  • Sound effects: Integrate subtle sound effects for dice rolls, holding score, and winning/losing.
  • More advanced styling: Explore CSS animations for dice rolls or player transitions beyond simple class toggles.
